Token Counting

Generally, each token key authorizes a specific number of tokens. Depending on the product and its version, counting can mean different things. For example, in ILOG Server 4.1, one token is required for each connection, and the Usage column of the ilmlist command output lists the connection id. If your product documentation does not define how to count tokens, then one token is required for each user of each machine, and the Usage column remains blank in the ilmlist (or ilmlist.exe) command output.

In the following example, each ILOG Server connection (e.g. 192.127.05.3121.6) is authorized by one specific token. As there are currently 5 connections, 5 tokens are being used. Only five tokens can be used freely by any given user.

For example, Joe runs 2 ILOG Views 3.1 processes on the machine sol78. ILOG Views 3.1 documentation does not specify token counting. Therefore, the token is assigned to the machine hostname. Liz uses one token and Bob uses another. As a result, the three tokens can be used freely.
